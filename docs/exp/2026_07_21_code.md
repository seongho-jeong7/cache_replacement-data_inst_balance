# 2026-07-21 코드 노트: L2C Partition Replacement와 Prefetcher 정리

이 문서는 `2026_07_21_analysis.md`에서 정리한 토론을 바탕으로, `ChampSim_FDIP`에 반영한 코드 변경을 patch note 형태로 기록한다. 핵심 목적은 L2C instruction/data hard partition 모델에서 replacement와 prefetcher가 실험 의도를 흐리지 않도록 정리하는 것이다.

## 목적

L2C partition 실험은 하나의 `cpu0_L2C`를 instruction/data origin에 따라 나눠 쓰는 모델이다. 따라서 `2i6d`, `4i4d`, `6i2d` 같은 설정에서는 line fill과 victim 선택이 반드시 자기 partition 안에서만 일어나야 한다.

기존 구현은 invalid way는 partition 내부에서 찾았지만, partition이 꽉 찬 경우 replacement policy가 전체 set에서 victim을 먼저 고른 뒤 partition 밖이면 첫 way로 fallback했다. 이 방식은 동작은 하지만 LRU/RRIP state를 왜곡할 수 있다.

또한 L2C prefetcher(`ip_stride`)는 I/D origin을 전달받지 못한다. L2C prefetcher가 instruction-origin access를 보고 prefetch를 발행해도, 새 prefetch packet의 `is_instr_fetch` 기본값은 false라 data-origin처럼 취급될 수 있다. 따라서 partition 실험에서는 L2C prefetcher가 data partition을 오염시킬 위험이 있다.

## 수정 방향

- L2C replacement victim 후보를 처음부터 partition 내부 way 범위로 제한한다.
- replacement API에 `[victim_way_begin, victim_way_end)`를 전달한다.
- built-in replacement policy(`lru`, `srrip`, `drrip`, `ship`, `random`)가 모두 후보 range 안에서만 victim을 고르도록 수정한다.
- 기존 partition 밖 victim fallback 로직을 제거한다.
- L2C prefetcher를 `ip_stride`에서 `no`로 바꿔 I/D origin 오염을 제거한다.
- 빌드는 수행하지 않았다. 다음 실험 시작 전에 새 binary를 빌드해야 한다.

## Patch 1: L2C Victim 후보 범위 계산

기존에는 victim이 partition 밖으로 나오면 fallback으로 보정했다.

```diff
-  [[nodiscard]] long partition_fallback_way(bool is_instr_fetch) const;
+  [[nodiscard]] std::pair<long, long> l2c_victim_way_range(bool is_instr_fetch) const;
```

새 코드에서는 `cpu0_L2C` partition 상태와 request origin을 보고 replacement 후보 range를 만든다.

```diff
+auto CACHE::l2c_victim_way_range(bool is_instr_fetch) const -> std::pair<long, long>
+{
+  if (!l2c_id_partition || NAME != "cpu0_L2C")
+    return {0, static_cast<long>(NUM_WAY)};
+
+  const auto instr_ways = static_cast<long>(std::min(l2c_instruction_ways, NUM_WAY));
+  const auto data_ways = static_cast<long>(std::min(l2c_data_ways, NUM_WAY));
+  if (instr_ways + data_ways != static_cast<long>(NUM_WAY))
+    return {0, static_cast<long>(NUM_WAY)};
+
+  return is_instr_fetch ? std::pair{0L, instr_ways} : std::pair{instr_ways, instr_ways + data_ways};
+}
```

의미:

- `shared` 또는 비정상 partition 설정이면 기존처럼 전체 way를 후보로 사용한다.
- `2i6d`라면 instruction은 `[0,2)`, data는 `[2,8)`만 victim 후보가 된다.
- `0i8d`/`8i0d`의 0-way side는 L2C bypass 경로를 타므로 victim 후보를 만들지 않는다.

## Patch 2: Fallback 제거 및 후보 range 전달

기존 fill path는 전체 set에서 victim을 고르고 partition 밖이면 첫 way로 보정했다.

```diff
-    auto victim_way = impl_find_victim(fill_mshr.cpu, fill_mshr.instr_id, get_set_index(fill_mshr.address), &*set_begin, fill_mshr.ip, fill_mshr.address,
-                                       fill_mshr.type);
-    if (!way_matches_l2c_partition(victim_way, fill_mshr.is_instr_fetch))
-      victim_way = partition_fallback_way(fill_mshr.is_instr_fetch);
+    const auto [victim_way_begin, victim_way_end] = l2c_victim_way_range(fill_mshr.is_instr_fetch);
+    auto victim_way = impl_find_victim(fill_mshr.cpu, fill_mshr.instr_id, get_set_index(fill_mshr.address), &*set_begin, fill_mshr.ip, fill_mshr.address,
+                                       fill_mshr.type, victim_way_begin, victim_way_end);
+    assert(victim_way_begin <= victim_way);
+    assert(victim_way < victim_way_end);
+    assert(way_matches_l2c_partition(victim_way, fill_mshr.is_instr_fetch));
```

의미:

- replacement policy가 애초에 자기 partition 내부만 보고 victim을 선택한다.
- partition 밖 victim을 첫 way로 바꾸는 편향이 사라진다.
- hard partition 모델과 replacement state가 더 일관된다.

## Patch 3: Replacement API 확장

replacement module wrapper는 새 candidate range signature를 먼저 찾고, 기존 signature도 fallback으로 유지한다.

```diff
-    virtual long impl_find_victim(uint32_t triggering_cpu, uint64_t instr_id, long set, const BLOCK* current_set, champsim::address ip,
-                                  champsim::address full_addr, access_type type) = 0;
+    virtual long impl_find_victim(uint32_t triggering_cpu, uint64_t instr_id, long set, const BLOCK* current_set, champsim::address ip,
+                                  champsim::address full_addr, access_type type, long victim_way_begin, long victim_way_end) = 0;
```

```diff
+    if constexpr (replacement::has_find_victim<decltype(r), uint32_t, uint64_t, long, const BLOCK*, champsim::address, champsim::address, access_type, long,
+                                               long>)
+      return return_type{r.find_victim(triggering_cpu, instr_id, set, current_set, ip, full_addr, type, victim_way_begin, victim_way_end)};
```

의미:

- built-in replacement는 새 signature를 사용한다.
- 기존 style의 외부 replacement module도 wrapper fallback 덕분에 당장 깨지지 않는다.
- 다만 partition-aware replacement를 원한다면 새 signature를 구현하는 것이 맞다.

## Patch 4: LRU/RRIP/SHIP/Random의 후보 범위 적용

LRU는 후보 range 안에서만 가장 오래 사용되지 않은 way를 고른다.

```diff
-  auto begin = std::next(std::begin(last_used_cycles), set * NUM_WAY);
-  auto end = std::next(begin, NUM_WAY);
+  auto set_begin = std::next(std::begin(last_used_cycles), set * NUM_WAY);
+  auto begin = std::next(set_begin, victim_way_begin);
+  auto end = std::next(set_begin, victim_way_end);

   auto victim = std::min_element(begin, end);
-  return std::distance(begin, victim);
+  return std::distance(set_begin, victim);
```

SRRIP/DRRIP/SHIP도 동일하게 candidate range 안에서만 max RRPV victim을 찾고, RRPV aging도 그 range 안에서만 수행한다.

```diff
-  auto begin = std::next(std::begin(rrpv), set * NUM_WAY);
-  auto end = std::next(begin, NUM_WAY);
+  auto set_begin = std::next(std::begin(rrpv), set * NUM_WAY);
+  auto begin = std::next(set_begin, victim_way_begin);
+  auto end = std::next(set_begin, victim_way_end);

   auto victim = std::max_element(begin, end);
   if (auto rrpv_update = maxRRPV - *victim; rrpv_update != 0)
     for (auto it = begin; it != end; ++it)
       *it += rrpv_update;
```

Random은 매 호출마다 candidate range 기반 distribution을 만든다.

```diff
-  return dist(rng);
+  std::uniform_int_distribution<long> candidate_dist(victim_way_begin, victim_way_end - 1);
+  return candidate_dist(rng);
```

의미:

- `2i6d`: instruction victim 후보는 way 0~1, data victim 후보는 way 2~7.
- `4i4d`: instruction/data가 각각 4 way 범위 안에서만 victim 선택.
- `6i2d`: data victim 후보가 way 6~7로 제한된다.
- shared mode는 `[0, NUM_WAY)`이므로 기존 동작과 동일하다.

## Patch 5: L2C Prefetcher Off

L2C prefetcher는 I/D origin을 알지 못하기 때문에 partition 실험에서는 오염원이 될 수 있다. 따라서 현재 기준 config에서 L2C prefetcher를 끈다.

```diff
     "L2C": {
         ...
-        "prefetcher": "ip_stride"
+        "prefetcher": "no"
     },
```

의미:

- L2C partition 실험에서 capacity/lookup/replacement 효과를 prefetcher origin 오염 없이 본다.
- L1D prefetcher는 `ip_stride`로 유지한다.
- L1I config prefetcher는 `no`지만, FDIP는 별도 frontend path에서 L1I prefetch를 발행한다.

## 현재 Policy 요약

| Level | Prefetcher | Replacement |
|---|---|---|
| L1I | `no` | `lru` default |
| L1D | `ip_stride` | `lru` default |
| L2C | `no` | `lru` default |
| LLC | `no` | `lru` |
