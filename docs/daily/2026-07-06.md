- 빌드 결과는 `html/` 폴더에 저장되도록 했다.

- `make html` 중 `ChampSim_DPC4/vcpkg` 내부 문서가 함께 읽히는 문제가 있어 `conf.py`의 `exclude_patterns`에 `ChampSim_DPC4`, `ChampSim_FDIP`, `outputs`를 추가했다.

문서 서버는 `html/`을 9179 포트로 열어 확인했다. 터미널이 끊겨도 유지되도록 `screen` 기반 실행을 사용했다.

## ChampSim Workflow

ChampSim 실행 흐름을 루트 스크립트로 정리했다.

- `scripts/setup_champsim.sh`를 추가해 submodule 초기화와 vcpkg 설치 명령을 정리했다.
- `scripts/run.sh`를 만들고 `-b`, `-t`, `-p`, `-s` 옵션을 추가했다.
- `-b`는 configure/build, `-t`는 trace 실행, `-p`는 병렬 실행 수 지정, `-s`는 최신 결과 summary 출력으로 정의했다.
- 기본 병렬 실행 수는 16으로 설정했다.
- trace는 `traces/gtrace_v2`를 기본 대상으로 삼았다.
- 결과는 `outputs/YYMMDD_HHMM/<trace_set>/<trace_group>/...log` 구조로 저장하도록 변경했다.
- 전체 실행 로그는 각 날짜 폴더의 `run.log`에 저장하고, 화면에는 에러 중심으로 출력하도록 정리했다.

초기에는 루트 `config/config_default.json`을 사용했지만, 이후 `config/` 폴더를 제거하고 `CHAMPSIM_DIR/champsim_config.json`을 직접 사용하는 방식으로 바꿨다. 이 방식이 ChampSim, DPC4, FDIP처럼 여러 ChampSim 변형을 다룰 때 더 자연스럽다.

## Output Parsing

실행 결과를 CSV와 표 형태로 정리하는 파서를 만들었다.

- `parser/parse_outputs.py`에서 각 trace 로그를 읽어 `metrics.csv`를 생성하도록 했다.
- IPC, branch MPKI, L1D/L2C/LLC/STLB MPKI, off-chip traffic 등을 추출했다.
- 실패한 trace도 CSV에 `failed` 또는 `incomplete` 상태로 남기도록 했다.
- `parser/summary.py`를 추가해 `metrics.csv`를 group 단위 표로 요약했다.
- `scripts/run.sh -s`를 실행하면 최신 output 폴더의 `metrics.csv`를 자동으로 찾아 summary를 출력하도록 했다.

이 과정에서 최신 output 폴더에 `metrics.csv`가 없으면 `-s`가 이전 폴더를 분석하는 문제가 있었다. 이후 최신 폴더에 metrics가 없을 때 자동 생성하도록 정리했다.

## FDIP Setup

기존 `ChampSim`과 `ChampSim_FDIP`를 비교했다. 처음에는 ChampSim 버전 차이가 섞여 있었기 때문에 기존 diff 문서를 제거하고, `ChampSim_DPC4`와 `ChampSim_FDIP` 기준으로 다시 비교 문서를 작성했다.

FDIP 쪽은 외부로 빼지 않고 `ChampSim_FDIP/` 내부에서 먼저 테스트했다.

- FDIP off와 FDIP on FTQ16을 1개 workload로 실행했다.
- FDIP 분석 스크립트인 `script/fdip_cover.py`를 확인했다.
- 분석 결과 파일은 `ChampSim_FDIP/fdip_test/`에 저장하는 방향으로 조정했다.
- 이후 `ChampSim_FDIP/`를 루트 프로젝트에 일반 폴더로 추가하고 커밋했다.

FDIP 실행을 루트 `scripts/run.sh`와 통합하기 위한 작업도 시작했다.

- `scripts/run.sh`의 기본 `CHAMPSIM_DIR`를 `ChampSim_FDIP`로 설정했다.
- `-f <num>` 옵션을 추가해 FDIP FTQ size를 지정할 수 있게 했다.
- `-f 0`은 FDIP off, `-f 16`은 FTQ16 설정으로 사용할 수 있다.
- 출력 파일명과 config signature에 `ftq0`, `ftq16`처럼 FTQ 크기가 포함되도록 했다.
- FDIP 로그의 `L1I Demand Access Breakdown`을 `metrics.csv`에 포함하도록 parser 확장을 진행했다.
- summary 표에도 `FDIP Cov`, `L1I Miss` 컬럼을 추가했다.

FDIP 빌드 중 `fmt/core.h`를 찾지 못하는 문제가 있었다. 원인은 `ChampSim_FDIP/vcpkg_installed/`가 준비되지 않은 것이었다. 이를 위해 `scripts/setup_champsim_fdip.sh`를 추가했고, 기존 `ChampSim` 또는 `ChampSim_DPC4`의 vcpkg를 이용해 `ChampSim_FDIP/vcpkg.json` 기반 의존성을 설치하는 방향으로 정리했다.
